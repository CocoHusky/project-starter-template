# License Templates

This template repo does not force one license onto every future project.

When starting a new project, choose the license that fits that project, then copy one of these files to the repository root as `LICENSE`:

```text
LICENSES/MIT.txt          permissive, simple, common for small tools and examples
LICENSES/Apache-2.0.txt   permissive, includes explicit patent language
```

## Typical choices

Use `MIT` when you want a short permissive license for small projects, examples, utilities, demos, and simple open-source tools.

Use `Apache-2.0` when you want a permissive license with more explicit patent protection language, often useful for larger software projects, firmware, libraries, or projects that may be reused commercially.

## How to apply one

Copy the selected license file to the root of the new project:

```sh
cp LICENSES/MIT.txt LICENSE
```

or:

```sh
cp LICENSES/Apache-2.0.txt LICENSE
```

Then update the copyright line.
