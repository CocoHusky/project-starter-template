# Project Starter Template

A small, clean starter template for new GitHub projects.

Use this when you want a repo that starts organized without being locked into a specific language, framework, or build system.

## What is included

- A simple README structure
- A practical `.gitignore`
- A pull request template
- A bug report issue template
- A contributing guide
- A security policy
- MIT and Apache-2.0 license templates

## Start a new project

1. Update this README with the project name and purpose.
2. Add the actual project files.
3. Choose a license from `LICENSES/` if the project should be open source.
4. Keep private data, credentials, API keys, and local machine files out of Git.
5. Open pull requests with a short summary, changes list, and test plan.

## Suggested project structure

Use only what the project needs:

```text
src/      source code
docs/     notes and documentation
tests/    tests or validation files
assets/   images, diagrams, or screenshots
```

## License choice

This template does not force one license onto every generated project.

For a new project, copy one license template to the repo root as `LICENSE`:

```sh
cp LICENSES/MIT.txt LICENSE
```

or:

```sh
cp LICENSES/Apache-2.0.txt LICENSE
```

Then update the copyright line.
